## Phase 2 — Storage

**T03 — Initialise SQLite database**
In `Database.swift`, open (or create) `highlights.db` in `~/Library/Application Support/KindleWall/`. Run the `CREATE TABLE IF NOT EXISTS` statements for `books` and `highlights` exactly as specced. Expose a shared `DatabaseQueue` instance.

**T04 — Implement book upsert**
Write a function `upsertBook(_ book: Book) -> UUID` that inserts a new row into `books` or, if `(title, author)` already exists, returns the existing `id`. Uses the `UNIQUE(title, author)` constraint with `INSERT OR IGNORE`.

**T05 — Implement highlight insert with dedupe**
Write a function `insertHighlightIfNew(_ highlight: Highlight)` that computes the `dedupeKey`, checks if it already exists in `highlights`, and inserts only if not present. Never throws on duplicate — silently skips.

**T06 — Implement book enable/disable toggle**
Write a function `setBookEnabled(id: UUID, enabled: Bool)` that updates `isEnabled` in the `books` table. When a book is toggled **on**, also set `lastShownAt = NULL` for all its highlights. When toggled **off**, no highlight changes needed.

**T07 — Implement highlight query for wallpaper (COUNT + OFFSET)**
Write a function `pickNextHighlight() -> Highlight?` that:
1. Counts highlights where `bookId IN (enabled books) AND lastShownAt IS NULL`
2. Picks `LIMIT 1 OFFSET Int.random(in: 0..<count)`
3. If count is 0, resets `lastShownAt = NULL` for the entire active pool, then repeats
Returns `nil` only if the active pool is empty (no enabled books with highlights).

**T08 — Implement `markHighlightShown(id: UUID)`**
Write a function that sets `lastShownAt` to the current ISO8601 timestamp (with timezone offset) for a given highlight ID.

**T09 — Implement book list query**
Write a function `fetchAllBooks() -> [Book]` that returns all rows from `books`, sorted alphabetically by title, with a derived `highlightCount` via a `COUNT` subquery or join.

**T10 — Implement highlight count query**
Write a function `totalHighlightCount() -> Int` returning the total row count of `highlights`. Used for the "N highlights in library" display.

## Phase 3 — Parsing

**T11 — Implement `ClippingsParser`: chunk splitting**
In `ClippingsParser.swift`, write the first stage of parsing: normalize `\r\n` → `\n`, split on lines that equal `==========` when trimmed, discard empty chunks. Return an array of raw string chunks. No field extraction yet — just clean splitting.

**T12 — Implement `ClippingsParser`: per-chunk field extraction**
For each chunk from T11: skip leading blank lines, find the title line (first non-blank), find the metadata line (first line starting with `- Your `), skip chunks where metadata doesn't contain `Your Highlight`, extract quote body (lines after first blank line following metadata), skip if body is empty.

**T13 — Implement title and author cleaning**
Write a function `cleanTitleAndAuthor(from titleLine: String) -> (title: String, author: String)` covering both formats: standard `Book Title (Author Name)` and verbose sideloaded `Book Title -- Author;Editor -- ...`. Extract title before first ` --` and author from final `(...)` group or first `--` segment as fallback.

**T14 — Implement date parsing**
Write a function `parseKindleDate(_ string: String) -> Date?` using `en_US_POSIX` locale, system timezone, and format `"EEEE, MMMM d, yyyy h:mm:ss a"`. Returns `nil` on failure. Track failures via a `parseErrorCount` counter. Never drop an entry due to date failure.

**T15 — Implement dedupeKey computation**
Write a function `computeDedupeKey(bookId: UUID, location: String?, quoteText: String) -> String` implementing the primary and fallback key logic: lowercase + whitespace-collapse both location and first 50 chars of text, join with `"|"` separators, prefix with `bookId`.

**T16 — Wire full parse pipeline**
Write the top-level function `parseClippings(fileURL: URL) -> (highlights: [Highlight], books: [Book], parseErrorCount: Int)` that calls T11–T15 in sequence and returns fully populated model objects ready for database insertion.

## Phase 4 — Import

**T17 — Implement `ImportCoordinator`**

This is the single entry point for both auto and manual import.

**T18 — Implement `VolumeWatcher`: known path check**
In `VolumeWatcher.swift`, write `findClippingsFile(on volume: URL) -> URL?` that tries the three known paths (`/documents/My Clippings.txt`, `/Documents/My Clippings.txt`, `/My Clippings.txt`) in order and returns the first hit.

**T19 — Implement `VolumeWatcher`: bounded shallow search**
Extend `findClippingsFile` with the fallback traversal: max depth 3, hard cap of 2000 directories and 1 second elapsed, skip hidden/system dirs (`.` prefix, `.Spotlight-V100`, `.Trashes`, `.fseventsd`, `.app` packages). Return immediately on a `documents`-path match; otherwise return best match after scan completes.

**T20 — Implement `VolumeWatcher`: USB mount listener**
Register for `NSWorkspace.didMountNotificationName`. On each mount event, call `findClippingsFile` (T18 + T19). If found, call `importFile` (T17) and update the import status in `AppState`.

## Phase 6 — Scheduling

**T25 — Implement `WallpaperScheduler`**
In `WallpaperScheduler.swift`, implement a scheduler that:
- On init and every 60 seconds, checks the active schedule setting from `UserDefaults`
- **Daily:** if `now >= todayScheduledTime` and `lastChangedAt < todayScheduledStart`, call `rotateWallpaper()`
- **Every 30 min:** if `now - lastChangedAt >= 30 * 60`, call `rotateWallpaper()`
- **On launch:** call `rotateWallpaper()` once on app start
- **Manual:** do nothing (rotation only via "Next Quote" button)

### ✅ H4 — Non-blocking runtime log noise (`fopen` cache invalidation)
Reported by user during app runtime; appeared non-fatal.

Observed logs:
- `fopen failed for data file: errno = 2 (No such file or directory)`
- `Errors found! Invalidating cache...`

Root cause identified:
- Repeated background-image decode attempts could hit missing/unreadable/invalid files from persisted paths.
- Repeated decode attempts produced noisy lower-level framework messages.

Implemented fix:
- Added shared `BackgroundImageLoader` boundary with explicit outcomes:
  - `noImageConfigured`
  - `missingFile`
  - `unreadableFile`
  - `decodeFailed`
  - `success`
- Added one-time warning dedupe per `(failure kind, file path)` to avoid repeated runtime log spam.
- Added in-memory decode cache keyed by path + file identity (`mtime`, `size`) with invalidation on file change.
- Routed both settings-preview loading and wallpaper-generation loading through the same loader boundary.

Verification:
- Added deterministic `scripts/verify_h4.sh` coverage for missing/unreadable/decode-failed/success outcomes.
- Added coverage that warnings are deduped and cache invalidates when file identity changes.
- Added wallpaper-generator fallback-path coverage when background decode fails.
