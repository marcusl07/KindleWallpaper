# KindleWall — v1 Spec

## Overview

A lightweight macOS menu bar app that parses Kindle highlights from `My Clippings.txt`, overlays them as text on a user-picked background image, and sets the result as the desktop wallpaper. The wallpaper rotates through the user's highlight pool on a configurable schedule.

## Core Features (v1)

### 1. Highlight Import

#### Primary — Auto-sync on USB mount
- App listens for new disk volumes being mounted via `NSWorkspace.shared.notificationCenter` / `NSWorkspace.didMountNotificationName`
- When a new volume is mounted, attempt to locate `My Clippings.txt` using the following strategy:
  1. **Known path check first** — try these case variants in order:
     - `<volume>/documents/My Clippings.txt`
     - `<volume>/Documents/My Clippings.txt`
     - `<volume>/My Clippings.txt`
  2. **Bounded shallow search fallback** — if none of the known paths match, do a directory traversal of the volume with the following rules:
     - Max depth: 3
     - Hard cap: stop after visiting 2000 directories or after 1 second, whichever comes first
     - Skip hidden/system directories: `.Spotlight-V100`, `.Trashes`, `.fseventsd`, any dir starting with `.`, and any `.app` package contents
     - Search for filename `My Clippings.txt` (case-insensitive)
     - **Match preference:** if a match is found under a path segment named `documents` (case-insensitive), return it immediately. Otherwise, record the first match found and continue the scan; return the best (documents-preferring) match after the scan completes.
- If found, app automatically reads, parses, and merges new highlights
- No user interaction required
- No macOS notifications under any circumstance — success/failure is surfaced in the Settings window only

#### Fallback — Manual file import
- User can manually import a `My Clippings.txt` file via the settings window (open file picker)
- Same parsing and deduplication logic applies

#### Import Status (shown in Settings window, Import section)
- On success: `"Last synced: [date/time] — [N] new highlights added"` or `"Library up to date"` if nothing new
- On failure: `"Import failed: [short reason]"` displayed in red — e.g. file unreadable, no valid highlights parsed
- Status persists until the next import attempt

### 2. Parsing `My Clippings.txt`

The file uses `==========` as a separator between entries. The **actual** structure of each entry is:

```
[Book Title] (Author Name)
- Your Highlight on page X | Location X-X | Added on [Day], [Month] [D], [YYYY] [H:MM:SS AM/PM]
[blank line]
[highlight text — may be multiple lines]
==========
```

**Parsing approach (for Codex):**
1. Normalize line endings: replace all `\r\n` with `\n`
2. Split on lines that equal `==========` when trimmed (do not rely on surrounding newline characters)
3. Trim leading and trailing whitespace from each chunk; discard empty chunks
4. For each chunk, split into lines and skip any leading blank lines
5. **First non-blank line** = title/author line
6. **Find the metadata line** = first line starting with `- Your ` — this is more resilient than assuming line index 1
7. **Skip** any entry where the metadata line does not contain `Your Highlight` — v1 ignores notes and bookmarks
8. **Quote body** = all lines after the first blank line that follows the metadata line, joined with a space; trim result
9. **Skip** any entry where the quote body is empty after trimming
10. Extract fields with regex from title line and metadata line

**v1 scope:** English-only metadata line format. No localisation handling needed.

**MVP import strategy:** Parse the full file on every import and deduplicate against the existing database. No incremental offset tracking. Simple and reliable at expected scale (~10k highlights max).

**Title/author cleaning:**
- Standard format: `Book Title (Author Name)` — extract author from the final `(...)` group
- Sideloaded/verbose format: `Book Title -- Author;Editor -- Publisher -- ...` — take everything before the first ` --` as the title; extract author from the final `(...)` group if present, otherwise take the segment immediately after the first ` --`

**Fields to extract per highlight:**
- `quoteText` — full highlight body, trimmed
- `bookTitle` — cleaned title string
- `author` — cleaned author string
- `location` — raw location string (e.g. `"212-213"`); may be absent for some sideloaded books — store as optional
- `dateAdded` — parsed `Date` object; use `en_US_POSIX` locale and system timezone when parsing
  - Expected format: `"EEEE, MMMM d, yyyy h:mm:ss a"` (e.g. `"Wednesday, May 7, 2025 11:04:04 PM"`)
  - Store as ISO8601 string with timezone offset in the DB (e.g. `"2025-05-07T23:04:04+00:00"`)
  - If parsing fails: set `dateAdded` to `NULL` and continue importing — **never drop an entry due to date parse failure**; increment a `parseErrorCount` counter for diagnostics
- `id` — stable UUID generated at import time

**Deduplication key (for Codex):**
- Primary: `bookId + "|" + locationNormalized + "|" + first50CharsNormalized` where location and text are lowercased and whitespace-collapsed
- Fallback (when location is missing): `bookId + "|" + first50CharsNormalized`
- Including `bookId` prevents cross-book collisions where two books share the same location range and opening text
- On import, compute key for each parsed entry; skip if key already exists in the database

### 3. Highlight Storage

**SQLite for highlight data** (via `GRDB.swift` or direct SQLite3):
- Stored in `Application Support/KindleWall/highlights.db`
- Expected scale: up to ~10k highlights — SQLite is well-suited and enables future filtering/querying by book
- Schema:

```sql
CREATE TABLE books (
    id          TEXT PRIMARY KEY,            -- UUID string
    title       TEXT NOT NULL,
    author      TEXT NOT NULL,
    isEnabled   INTEGER NOT NULL DEFAULT 1,  -- 1 = included in rotation pool
    UNIQUE(title, author)
);

CREATE TABLE highlights (
    id          TEXT PRIMARY KEY,       -- UUID string
    bookId      TEXT NOT NULL,          -- references books.id
    quoteText   TEXT NOT NULL,
    bookTitle   TEXT NOT NULL,          -- denormalised for display convenience
    author      TEXT NOT NULL,          -- denormalised for display convenience
    location    TEXT,                   -- nullable
    dateAdded   TEXT,                   -- ISO8601; nullable if parse fails
    lastShownAt TEXT,                   -- ISO8601; NULL until first shown
    dedupeKey   TEXT NOT NULL UNIQUE    -- locationNormalized+first50 or fallback
);
```

- On import: upsert each book by `(title, author)` unique index, then insert highlights referencing `books.id` via `bookId`
- `bookTitle` and `author` are denormalised onto highlights for display convenience — no JOIN needed to render a wallpaper

**UserDefaults for small settings only:**
- Rotation schedule selection (enum: manual / daily / onLaunch / every30min)
- Daily rotation time: hour + minute (default 9:00 AM)
- Last wallpaper change timestamp
- Background image path (in `Application Support`)
- Note: shuffle state is no longer stored in UserDefaults — see Quote Order

---

### 4. Background Image

- User picks one image file via a file picker (any common format: JPG, PNG, HEIC)
- Image is stored by copying it into `Application Support` so the original can be moved/deleted freely
- v1: one background image only, no multiple images or rotation of backgrounds
- **Default (no image set):** render quote text on a solid black background at the screen's native resolution

---

### 5. Wallpaper Generation

When it's time to set a new wallpaper:

1. Pick the next highlight from the active pool (random, no repeats — see Quote Order)
2. Load the background image
3. Render quote text on top using system font (SF Pro / `.systemFont`)
4. Save the composited image as a PNG to a temp location in `Application Support`
5. Set the composited image as wallpaper on **all connected screens** by iterating `NSScreen.screens` and calling `NSWorkspace.shared.setDesktopImageURL(_:for:options:)` for each

**Text rendering defaults (v1):**
- Font: `NSFont.systemFont(ofSize: 36, weight: .regular)`
- Color: white
- Position: horizontally and vertically centered
- Semi-transparent dark overlay behind the full image (black at ~40% opacity) to ensure text legibility regardless of background image
- Author + book title rendered below the quote in a smaller font: `NSFont.systemFont(ofSize: 20, weight: .light)`, format: `"Book Title - Author"`
- For sideloaded books with verbose titles, clean the title by taking everything before the first ` --` occurrence
- Max text width: 70% of image width, word-wrapped

### 6. Rotation Schedule

User can configure when the wallpaper changes. Options:

| Option | Behaviour | Notes |
|---|---|---|
| Manual only | Only changes when user clicks "Next Quote" | Primary option |
| Daily at set time | Fires once per day at a user-configured time | Primary option |
| On app launch | Changes wallpaper every time the app starts | Secondary / testing |
| Every 30 minutes | Timer fires every 30 minutes while app is running | Secondary / testing |

- Setting persisted in `UserDefaults`
- Default: **Daily at 9:00 AM**
- **Daily schedule logic:** on app launch and every 60 seconds thereafter, check if `now >= todayScheduledTime` and `lastChangedAt < todayScheduledStart`; if both true, rotate once. This fires at most once per calendar day regardless of how many times the app launches.
- User can change the daily time via a time picker in Settings (hour/minute stored in UserDefaults)
- Timed rotation only works while the app is running — noted in UI as a muted caveat
- "On system restart" is dropped — "On app launch" covers the same intent cleanly

**Known gap being addressed in Phase 10 (T55):**
- If the daily scheduler tick occurs while a rotation is already in progress and `requestWallpaperRotation()` returns `false` (busy), that scheduled rotation can be dropped for the day.
- Phase 10 addresses this with retry or queue behavior so the scheduled rotation eventually runs.

### 7. Quote Order

- v1: **random**, no repeats until all highlights in the active pool have been shown, then cycle resets
- Active pool = all highlights where `bookId` references a book with `isEnabled = 1`
- **Implementation via `lastShownAt` column (no large UserDefaults list):**
  1. Count eligible highlights: `SELECT COUNT(*) FROM highlights WHERE bookId IN (enabled books) AND lastShownAt IS NULL`
  2. Pick a random offset: `randomIndex = Int.random(in: 0..<count)`
  3. Fetch: `SELECT * FROM highlights WHERE bookId IN (enabled books) AND lastShownAt IS NULL LIMIT 1 OFFSET randomIndex`
  4. If count is 0 (pool exhausted): reset `lastShownAt = NULL` for the entire active pool, then repeat from step 1
  5. After showing a highlight: set its `lastShownAt` to the current ISO8601 timestamp
  - Note: `ORDER BY RANDOM()` is acceptable for v1 but can be slow at scale; the COUNT + OFFSET approach above is preferred
- **When book selection changes (book toggled on or off):**
  - Do **not** wipe `lastShownAt` for the entire pool
  - When a book is toggled **on**: set `lastShownAt = NULL` for all its highlights so they become immediately eligible
  - When a book is toggled **off**: its highlights are simply excluded by the query — no changes needed
  - This preserves history for books that stay enabled and avoids re-showing recent quotes unnecessarily

### 8. Book Selection

A dedicated section in the settings window listing all books discovered from imported highlights. The user can include or exclude individual books from the rotation pool.

**Behaviour:**
- Every book found during parsing is added to the `books` table with `isEnabled = 1` by default
- The rotation pool only draws from highlights belonging to books where `isEnabled = 1`
- Toggling a book off immediately excludes its highlights from the pool — the cycle continues uninterrupted for remaining enabled books
- If all books are toggled off, show an inline warning: `"No books selected — wallpaper won't update"`

**UI (within Settings Window):**
- List of all books, each row showing: `[checkbox] Book Title — Author`
- Rows sorted alphabetically by title
- `Select All` / `Deselect All` buttons above the list
- Highlight count per book shown in muted text: e.g. `(42 highlights)`

**Known implementation issue (2026-02-22):**
- Current build shows intermittent interaction reliability in this section:
  - Per-book checkboxes may apply on first click, require repeated clicks, or ignore clicks.
  - `Select All` / `Deselect All` may sometimes no-op.
  - Additional observed pattern: only the first few rows (first ~3 checkboxes) respond reliably; deeper rows in the list may not respond.
- Next agent should treat this as a flakiness bug (event/state propagation) rather than a fully broken feature.

**Menu items:**

- Separator

**Sections:**

**Rotation Schedule**
- Label: `Change wallpaper:`
- Radio buttons or picker:
  - Manual only

**About**
- App name + version string

### 11. Phase 11 — Wallpaper Collection and UX Safeguards

**Background image collection (T56, T57, T58)**
- **T56 — Background image collection data model:** replace the single background image path in `ScheduleSettings` or `AppState` with a collection-based model, with migration so an existing single image becomes the first collection item.
- **T57 — Background image collection UI:** replace the Settings `Change Image...` flow with a managed backgrounds view (similar pattern to the Books window), including previews, add/remove controls, blocking removal of the last image, and prompting to add if empty. UI should try to follow the default macOS Wallpaper settings menu patterns.
- **T58 — Random background selection in rotation pipeline:** update wallpaper rotation to randomly select from the background image collection on each rotation. A single-image collection should behave like current v1 behavior. T54 cache invalidation must account for the selected image.

**Capitalize highlight text toggle (T59)**
- Add a Settings toggle that capitalizes the first letter of highlight text when it is lowercase. Default is off.

**macOS wallpaper cycling conflict warning (T60)**
- Add a small Settings notice telling users to disable macOS built-in wallpaper rotation to avoid conflicts with KindleWall-managed wallpaper updates.

## Data Model

```swift
struct Highlight: Identifiable {
    let id: UUID
    let bookId: UUID           // references Book.id
    let quoteText: String
    let bookTitle: String      // denormalised for display
    let author: String         // denormalised for display
    let location: String?      // optional — absent for some sideloaded books
    let dateAdded: Date?       // optional — nil if date parsing failed
    var lastShownAt: Date?     // nil until first shown; reset on cycle exhaustion
    // dedupeKey is a DB implementation detail, not surfaced in the Swift model
}

struct Book: Identifiable {
    let id: UUID
    let title: String
    let author: String
    var isEnabled: Bool        // controls inclusion in rotation pool
    let highlightCount: Int    // derived via COUNT query, not stored
}
```
